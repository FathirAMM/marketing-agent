Marketing Multi-AI-Agent System

Overview: A hierarchical multi-agent orchestration framework built using the Google Agent Development Kit (ADK). It leverages specialized agents to handle research, copywriting, and visual creation, coordinating them to execute brand-aligned campaigns. Deployed on Cloud Run.

Key Features:

  • Hub-and-Spoke Architecture: 'Marketing Coordinator' orchestrates specialized sub-agents (Researcher, Social Lead, Visualist).
  • Multi-Source RAG: Ingests data from websites, annual reports, and YouTube videos into a scalable Qdrant vector database.
  • Advanced Capabilities: Uses Gemini for reasoning/content and image generation with brand guardrails.
  • Production Ready: Fully scalable architecture with Docker and Google Cloud Run deployment.

Tech Stack: Python, Google ADK, Gemini, Qdrant, LangChain, OpenAI Embeddings, Deepseek OCR, Firecrawl, Ollama, groq, Cloud Run.

Customer Churn Prediction: End-to-End Production ML Pipeline

Overview: A production-grade machine learning system designed to predict customer churn in a banking context. It automates the entire ML lifecycle from data ingestion to model deployment, focusing on MLOps best practices, scalability, and reliability.

Key Technical Features:

  • Real-Time Streaming: Leverages Apache Kafka for processing customer events at scale, providing real-time churn predictions.
  • Scalable Orchestration: Uses Apache Airflow for complex workflows, including weekly model retraining and daily data processing.
  • Experiment Tracking: Integrates MLflow for rigorous experiment tracking, model versioning, and lifecycle management.
  • Quality Gates: Automated CI/CD validation for data drift and model performance.

Tech Stack: Python, XGBoost, PySpark, MLflow, Airflow, Kafka, Docker, AWS (S3, RDS).

Multimodal Chatbot for Annual Report Analysis

Overview: A comprehensive multimodal chatbot system designed to analyze and extract insights from annual financial reports. It supports processing text, tables, and images from PDFs and other sources using Retrieval-Augmented Generation (RAG).

Key Features:

  • Multimodal RAG Application: Analyzes text, complex tables, and visual elements (charts, graphs). Uses GPT-4 Vision and supports multiple LLMs.
  • Document Processing: Partitioning PDFs using 'unstructured', data injection from multiple sources, and Vector DB integration (FAISS, Pinecone).
  • Advanced Functionalities: Chat with PDF, structured Report Generation, Fine-tuning module, and RAG evaluation.

Backend: Flask, LangChain, OpenAI/Anthropic, FAISS/Pinecone.
Frontend: React.js, Tailwind CSS, Real-time chat, Markdown/LaTeX.

PPE Detection System

Overview: A Streamlit-based web application for real-time detection and analysis of Personal Protective Equipment (PPE) in videos. It leverages a custom YOLOE-11l-seg model to identify human presence and verify safety gear compliance.

Key Features:

  • Real-time PPE Detection: Identifies safety vests, helmets, gloves, goggles, and shoes.
  • Person Tracking & Analysis: Detects people in frames and evaluates their PPE compliance individually.
  • Interactive Dashboard: Side-by-side comparison, frame-by-frame analysis, and visual status indicators (✓/✘).

Tech Stack: Python, Streamlit, Ultralytics (YOLOE), Supervision, OpenCV, Docker.
